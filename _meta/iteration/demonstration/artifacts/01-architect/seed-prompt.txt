Build a single canonical URL-safe slugifier package for the content pipeline. Today two callers (the index builder and the link renderer) each roll their own slug logic and produce divergent slugs for the same title, dead-linking each other. Replace that drift with one package covering exactly three caller needs: (1) a core `slugify(input: string): string` in `src/slugify.ts` — lower-case ASCII, hyphen-separated words, numbers preserved, Latin accents normalised via NFD + combining-mark strip, non-Latin/emoji dropped, consecutive non-alphanumerics collapsed to one hyphen, leading/trailing hyphens trimmed, empty input returns empty string; (2) batch helpers in `src/batch.ts` — `slugifyMany(inputs: string[]): string[]` and `uniqueSlug(slug: string, taken: string[]): string` (returns the slug or the smallest `-N` suffix not in taken: foo -> foo, [foo] -> foo-2, [foo,foo-2] -> foo-3); (3) configurable options on `slugify` — export `type SlugifyOptions = { separator?: string; maxLength?: number }` and accept an optional second argument (custom separator; positive-integer length cap with trailing-separator re-trim); no-options behaviour unchanged. Tests live under `tests/` and must cover the edge cases (empty input, emoji, non-Latin scripts, consecutive separators, suffix disambiguation, both options). Out of scope: pluggable transliteration tables, locale-aware lowercasing, corpus-persisted uniqueness. Size this as a small, releasable initiative with one feature per caller need.
