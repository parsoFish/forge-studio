The headline: the final loop is closed

Before this work a cycle stopped at "PR merged + reflected" — no docs update, no version, no changelog, no release in any phase. Forge now drives work all the way to a published version, with the operator's existing Approve as the only gate and CI owning the irreversible tag/publish.

Opt-in per project via a declared releaseProcess; a project without one is byte-for-byte unchanged. Forge never runs tag/publish — it ships the CI workflow template. Terminal proof: release.json + a release.finalized event.
