# Unifier pr-description restore loop

## What happened

Cycle INIT-2026-06-19-framework-release-definition. Unifier ran 12 iterations. Primary work per iteration: detect `chore: drop forge scratch` had stripped `.forge/pr-description.md`, re-add with `git add -f`, commit, push. All 12 iterations passed quality gates; the actual substantive unifier work (UWI-1) completed in iteration 12 as a side effect.

Unifier cost: ~$0.31/iteration × 12 = ~$3.7 to restore one file repeatedly.

## Root cause

The `chore: drop forge scratch` automated commit targets scratch files under `.forge/` but does not preserve `pr-description.md`. Since `pr-description.md` is tracked (not gitignored) and generated by the reviewer, it should survive the scratch-drop. The scratch-dropper's exclude list is missing `pr-description.md`.

## Fix candidate

Add `.forge/pr-description.md` to the scratch-drop exclusion list (or to a dedicated gitattribute pattern). This is a forge machinery change — the orchestrator that issues the scratch-drop commit should skip `pr-description.md` unconditionally.
