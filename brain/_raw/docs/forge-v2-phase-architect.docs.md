# Phase: Architect

> *Human-in-the-loop.* Turns ideas into initiatives via interactive Claude Code skills.

## Purpose

Collaborate with the user during ideation sessions to update the project's roadmap and emit one or more **initiatives** — coherent, dependency-ordered collections of features that move a project to a desired state. Once an initiative is confirmed, it's queued for unattended execution; the next human touch is the review loop.

## Inputs

- User's free-form idea / brief / pain point.
- Project's existing roadmap (`projects/<name>/roadmap.md`).
- Brain knowledge (queried via `brain-query`).
- Past initiatives + retros for the project.

## Outputs

- Updated `projects/<name>/roadmap.md`.
- One or more `_queue/pending/<initiative-id>.md` manifests with frontmatter (initiative_id, project, created_at, iteration_budget, cost_budget_usd, features array with depends_on edges).
- The markdown initiative spec (LLM-Council-confirmed brief).

## Skills

- `skills/architect/SKILL.md` — primary interactive skill.
- `skills/architect-llm-council/SKILL.md` — chained-reviewer pattern (CEO/eng/design/DX critics) auto-resolves mechanical questions and surfaces only taste decisions to the user. Inspired by gstack `/autoplan`.

## Success signals

- **Roadmap quality:** initiatives have explicit dependencies and clear acceptance criteria.
- **Council convergence:** the LLM Council reaches agreement on most points without escalating to user.
- **Downstream execution:** initiatives generated by the architect have higher unattended completion rate than ad-hoc-written ones.
- **Time-to-initiative:** human spends bounded time per ideation session.

## Known failure modes

- **Agent over-reaches** — turns one idea into 50 features. Architect prompt caps initiative size and prefers small, releasable initiatives.
- **Vague acceptance criteria** — propagates downstream and breaks developer loop. Council includes a critic specifically watching for this.
- **Missing dependencies** — leads to parallel work-item collisions. Council includes a dependency-graph critic.
